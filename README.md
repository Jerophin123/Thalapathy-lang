## <span style="color:#E32227">рҹҗҜ What Is</span> <span style="color:#FFC300">THALAPATHY?</span>

THALAPATHY (working title: *"Leo of Languages"*) is an independent experimental programming language that blends serious language-design engineering (Г  la C, C++, C#, Java, Python, JavaScript) with unapologetic Vijay-movie energy. Think of it as what would happen if a compiler theory textbook and a mass-hero interval block had a baby, and that baby grew up to have opinions about static typing.

- Write code in `.tvk` files.
- Compile it down to `.vijay` bytecode, stamped with the `VJAY` magic header.
- Get roasted, encouraged, or lectured by the compiler depending on your chosen `--mode`.
- Feel like the hero of your own second-half twist every time a build passes.

## <span style="color:#E32227">рҹ”Ҙ Technical</span> <span style="color:#FFC300">Overview</span>

THALAPATHY is not just vibes вҖ” under the *maasu* hood is a genuinely serious toolchain:

- **Core Engine** вҖ” Modern C++20, with a custom recursive-descent parser, dynamic dispatch resolver, static type checker, AST-walking interpreter, and a compiler that emits **VEAF 3** binary bytecode. No shortcuts, no duplicate-role gimmicks.
- **TDK (THALAPATHY Development Kit)** вҖ” a full environment manager (`tdk`), a source formatter (`thalafmt`), and a bytecode inspector (`vijaydump`). One kit to rule the whole shooting schedule.
- **Modular Imports (`sarkar`)** вҖ” resolve dependencies with `sarkar <module>;`, backed by full DFS cycle-detection and path-traversal safeguards. `sarkar` never lets a module go missing on your watch.
- **Object-Oriented Model:**
  - `varisu` вҶ’ single inheritance ("legacy," literally)
  - `kaththi` вҶ’ multiple interface contracts (sharp, precise, no blade left unused)
  - `paaru` / `maathu` вҶ’ property getter/setter blocks ("look" / "change")
  - Method overloading, dynamic dispatch, and operator overloading via `mersal operator`
- **Diagnostics** вҖ” caret-highlighted visual error messages with switchable personality via `--mode professional|mass|political-parody`.


## рҹӣ пёҸ CLI Usage Reference

Set `TDK_HOME` to your installation directory and add `TDK_HOME/bin` to your system `PATH`.

### Running, Verifying, & Building Code

```bash
# Run a script directly
thalapathy examples/hello/hello.tvk

# Semantic verification only ("check pannu, don't release yet")
thalapathy --check examples/oop/classes.tvk

# Compile source to .vijay VEAF bytecode
thalapathy --build examples/hello/hello.tvk
```

### Project Management with TDK

```bash
# Verify environment health вҖ” "system check panren"
tdk doctor

# Scaffold a new workspace project вҖ” "first day first show"
tdk new my-app
cd my-app

# Compile and execute project defined by thalapathy.toml
tdk build
tdk run
```

### Source Formatting & Bytecode Disassembly

```bash
# Format source files recursively in place вҖ” "makeover time"
thalafmt src/ --write

# Inspect compiled bytecode headers вҖ” "background verification"
vijaydump hello.vijay --header
```

## рҹ§© C++ Embedding API

TDK ships with a linkable static library (`libthalapathy_toolchain.a`) and public headers, so C++ applications can embed the compiler and interpreter directly вҖ” no need to wait for a sequel.

```cpp
#include <iostream>
#include <thalapathy/Toolchain.hpp>

int main() {
    auto result = thalapathy::Toolchain::check("main.tvk");
    if (result.success) {
        std::cout << "Verification successful!\n";
        return 0;
    }
    return 1;
}
```

Link downstream targets using CMake:

```cmake
find_package(THALAPATHY CONFIG REQUIRED)
target_link_libraries(my_app PRIVATE THALAPATHY::toolchain)
```
