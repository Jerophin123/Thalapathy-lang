> *"Once I commit, I never look back."* 🔥

THALAPATHY is an experimental general-purpose, multi-paradigm programming language. It blends modern programming language design with Vijay cinema culture, dialogue-style humour, and optional satire diagnostics — while shipping a real C++20 compiler, bytecode emitter, and interpreter you can run today.

## Quick start

```bash
# Build
cmake -S . -B build-release -DCMAKE_BUILD_TYPE=Release -G Ninja
cmake --build build-release --config Release

# Install (pick your prefix)
cmake --install build-release --prefix "D:\TDK-1.0.0"

# Configure environment (PowerShell example)
$env:TDK_HOME = "D:\TDK-1.0.0"
$env:PATH = "$env:TDK_HOME\bin;$env:PATH"

# Verify & run
tdk doctor
thalapathy examples/mass/full_language.tvk
```

## This is what THALAPATHY code looks like

The standard library is **Vijay cinema-native** — every module carries a film / dialogue identity, so a `.tvk` file is unmistakable. This runs as-is:

```bash
thalapathy examples/mass/full_language.tvk
```

```tvk
sarkar sivakasi;      // math   — "full firepower kanakku" 🎆
sarkar vaseegara;     // text   — dialogue processing 💬
sarkar kootam;        // collections — the crowd 👥
sarkar kootu;         // functional — map/filter/reduce 🔗

master Thalapathy varisu Nadigan {
    mersal init(string peru) { this.peru = peru; }
    mersal punch() -> string {
        thiruppi this.introduce() + " — once I commit, I never look back! 🔥";
    }
}

thalapathy aarambam {
    sollu("[sivakasi] mulai(169) =", sivakasi.mulai(169.0));      // sqrt → 13
    sollu("[vaseegara] thiruppu =", vaseegara.thiruppu("MASS"));  // → SSAM

    makkal adukku = kootam.adukku();          // Stack (LIFO)
    adukku.serthu("Ghilli");
    adukku.serthu("Master");
    sollu("[kootam] top =", adukku.eduthudu());                   // → Master

    makkal nums = [42, 7, 23, 4, 16];
    makkal evens = kootu.vadikattu(nums, kutty (n) { thiruppi n % 2 == 0; });
    sollu("[kootu] evens =", evens.join(", "));                   // → 42, 4, 16

    makkal hero = Thalapathy("Vijay");
    sollu(hero.punch());
}
```

> Idhu Java illa. Python illa. C++ illa. **Idhu THALAPATHY da.** 🔥

## THALAIVALAI — web development, THALAPATHY style

Serve HTML, CSS, JS, templates, and JSON REST APIs. Curl-verified demo: [`examples/web/02_alangaaram/`](examples/web/02_alangaaram/)

```tvk
sarkar thalaivalai;
sarkar bathil;

thalapathy aarambam {
    nanba app = thalaivalai.pudhu();
    app.sotthu("/alangaaram", "…/alangaaram");   // static CSS
    app.paarvaiIdam("…/paarvai");                // views

    app.kelu("/", kutty (kelvi) { thiruppi app.kaattu("veedu.html", onnumilla); });
    app.kelu("/api/heroes", kutty (kelvi) { thiruppi bathil.jilla([ /* … */ ]); });
    app.kelu("/hero/<peru>", kutty (kelvi) {
        thiruppi bathil.urai("Vanakkam " + kelvi.vazhiThunai("peru") + " nanba!");
    });

    app.oduda(8080);   // http://localhost:8080
}
```

Scaffold a new project: `tdk new-web my-app` · Full API: [docs/THALAIVALAI_WEB.md](docs/THALAIVALAI_WEB.md)

`paarvai` = views · `alangaaram` = CSS · `nadanam` = JS · route params `<name>`

## nanba & nanbi — immutable bindings

```tvk
nanba hero = "Vijay";                                      // single immutable binding
nanbi [peru, padam, varudam] = ["Vijay", "Ghilli", 2004]; // pattern binding
nanbi [first, _, ...rest] = [1, 2, 3, 4, 5];               // ignore + rest
vaathi nanbi [padam, varudam] ulla padangal { /* … */ }    // loop destructuring
```

Reference: [docs/NANBA_NANBI.md](docs/NANBA_NANBI.md) · Example: [`examples/mass/nanba_nanbi.tvk`](examples/mass/nanba_nanbi.tvk)

## Canonical syntax — prefer THALAPATHY over English

Legacy English spellings still compile; new code should use canonical forms. Migrate in bulk: `tdk thalapathify src/ -f`

| Legacy | Canonical THALAPATHY | Meaning |
|--------|---------------------|---------|
| `true` / `false` / `null` | `aama` / `illa` / `onnumilla` | Booleans & null |
| `this` / `super` | `naan` / `munnadi` | Instance & parent |
| `break` / `continue` | `interval` / `aduthu` | Loop control |
| `static` / `as` | `podhu` / `maathiko` | Static member & cast |
| `override` / `final` | `comeback` / `mudivu` | OOP modifiers |
| `in` (foreach) | `ulla` | `vaathi x ulla list` |

Audit & status: [docs/THALAPATHY_IDENTITY_AUDIT.md](docs/THALAPATHY_IDENTITY_AUDIT.md)

## Technical overview

- **Core engine** — Modern C++20: recursive-descent parser, dynamic-dispatch resolver, type checker, AST-walking interpreter, VEAF 3 bytecode compiler.
- **TDK** — All-in-one kit: `tdk` (project manager), `thalapathy` (driver), `thalafmt` (formatter), `vijaydump` (bytecode inspector).
- **Modules (`sarkar`)** — DFS cycle detection, path traversal safeguards, dotted imports.
- **OOP** — Single inheritance (`varisu`), interfaces (`kaththi`), properties (`paaru` / `maathu`), overloading, `mersal operator`, `comeback` override, `mudivu` final.
- **Pattern matching** — `yaaru <subject> { ivan … yaarumilla … }`.
- **Diagnostics** — Caret-highlighted errors; `--mode professional|mass|political-parody`.
- **Migration** — `tdk thalapathify` rewrites legacy English keywords to canonical THALAPATHY syntax.

Architecture deep-dive: [docs/TOOLCHAIN.md](docs/TOOLCHAIN.md) · Roadmap: [docs/ROADMAP.md](docs/ROADMAP.md)

```tvk
kaththi Nadigan { mersal punch() -> string; }

    mersal aarambam(string peru) { naan.peru = peru; }
    mersal punch() -> string { thiruppi "Vanakkam " + naan.peru + "!"; }
    comeback mersal punch() -> string { thiruppi "MASS — " + naan.peru; }
```

## Build & install

**Requirements:** CMake 3.20+, C++20 compiler (GCC 12+, Clang 14+, MSVC 2022+)

```bash
cmake -S . -B build-release -DCMAKE_BUILD_TYPE=Release -G Ninja
cmake --build build-release --config Release
cmake --install build-release --prefix "D:\TDK-1.0.0"
```

Set `TDK_HOME` to the install prefix and add `$TDK_HOME/bin` to `PATH`. Run `tdk doctor` to confirm.

## CLI reference

```bash
# Run, verify, compile
thalapathy examples/hello/hello.tvk
thalapathy --check examples/oop/classes.tvk
thalapathy --build examples/hello/hello.tvk
thalapathy --emit-ir script.tvk
thalapathy --repl

# Project management
tdk doctor
tdk new my-app
tdk new-web my-site          # THALAIVALAI scaffold
tdk build && tdk run

# Migration & formatting
tdk thalapathify src/ -f     # legacy → canonical keywords
thalafmt src/ --write
vijaydump hello.vijay --header
```

## Personality modes

Diagnostics adapt to your audience via `--mode`:

| Mode | Use when |
|------|----------|
| `professional` | CI, production builds, neutral error text |
| `mass` | Local dev — dialogue-flavoured messages (default in `tdk new`) |
| `political-parody` | Optional satire diagnostics — **fictional jokes only** |

```bash
thalapathy --mode professional script.tvk
thalapathy --mode mass examples/mass/full_language.tvk
```

Details: [docs/DIAGNOSTICS.md](docs/DIAGNOSTICS.md) · Satire guardrails: [docs/POLITICAL_SATIRE.md](docs/POLITICAL_SATIRE.md)

## C++ embedding

```cpp
#include <thalapathy/Toolchain.hpp>

int main() {
    auto result = thalapathy::Toolchain::check("main.tvk");
    return result.success ? 0 : 1;
}
```

```cmake
find_package(THALAPATHY CONFIG REQUIRED)
target_link_libraries(my_app PRIVATE THALAPATHY::toolchain)
```

## Editor support

The TDK ships a VS Code extension at `$TDK_HOME/tools/vscode/` (syntax highlighting, `.tvk` file association). Install from the unpacked folder or load from source in `editors/vscode-thalapathy/`.

## Contributing

Pull requests welcome. Keep the tribute respectful and the disclaimer intact.

1. Fork the repo and create a feature branch
2. Build & test: `cmake --build build-release && ctest --test-dir build-release`
